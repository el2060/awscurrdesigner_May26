1.Synopsis and Aims

This module aims to provide students with a thorough understanding of fundamental data structures and algorithm design principles that underpin modern software systems. Core topics include arrays, linked lists, stacks, queues, trees, heaps, hash tables, and graphs, together with the analysis of time and space complexity using Big-O notation. Students will examine classical algorithms for searching, sorting, and graph traversal, and apply systematic problem-solving techniques to design efficient computational solutions. Emphasis is placed on selecting appropriate data structures for given problem contexts, evaluating trade-offs between competing approaches, and implementing solutions in a modern programming language. On completion, students will be equipped to reason analytically about algorithmic efficiency and to apply these skills in subsequent software engineering and systems modules.

2.Learning Outcomes

On completion of the module, the students should be able to:

2.1
explain the properties and use cases of fundamental data structures:
  • arrays and dynamic arrays
  • linked lists (singly, doubly, circular)
  • stacks and queues
2.2
analyse the time and space complexity of algorithms using Big-O, Big-Ω, and Big-Θ notation
2.3
implement and evaluate sorting and searching algorithms:
  • bubble sort, insertion sort, merge sort, quicksort
  • linear search, binary search
2.4
apply tree and heap structures to organise and retrieve data efficiently:
  • binary trees and binary search trees
  • AVL trees and balancing principles
  • min-heap and max-heap
2.5
apply graph representations and traversal algorithms to solve network problems:
  • breadth-first search (BFS) and depth-first search (DFS)
  • Dijkstra's shortest-path algorithm
2.6
design and justify appropriate data structure choices for given software engineering scenarios
[Note — where applicable, implementation exercises require working code in Python or Java, with written justification of design decisions and complexity analysis.]

4.Teaching Methodology / Teaching & Learning Strategies
Problem-Based Learning
Students are presented with realistic computational problems drawn from industry contexts — scheduling systems, navigation engines, and search platforms. Working individually and in pairs, they apply data structure and algorithm knowledge to design, implement, and evaluate solutions. This approach develops analytical reasoning, code quality awareness, and the ability to justify design decisions with complexity evidence.
ICT-Enabled Learning
The module uses a blend of synchronous and asynchronous digital tools to support student learning. Resources include recorded micro-lectures, interactive visualisation tools for algorithm tracing, online auto-graded exercises, and discussion forums on the learning management system. Students are expected to complete pre-class preparation activities to maximise in-class problem-solving time.
Lab & Implementation Practicals
Hands-on lab sessions reinforce conceptual understanding through guided coding exercises in Python. Students implement each major data structure and algorithm from scratch, benchmark performance, and reflect on observed vs. theoretical complexity. Lab worksheets provide scaffolded tasks escalating from guided to independent implementation.

6.Plagiarism Policy

The Ngee Ann Polytechnic Plagiarism policy mandates that students must acknowledge the original author when they submit work. Failing to do so will constitute plagiarism.

All code submissions must be the student's own work. Use of AI-generated code without attribution and disclosure is not permitted. Students are expected to cite all external sources using the APA format and assume full responsibility for the academic integrity of their submitted work.
