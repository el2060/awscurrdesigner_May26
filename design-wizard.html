Area 1 — Section 1 of 2
Synopsis & Aims
AI Draft
This module provides students with a rigorous foundation in data structures and algorithm design — the core building blocks of efficient software systems. Students examine fundamental structures including arrays, linked lists, stacks, queues, trees, heaps, hash tables, and graphs, alongside classical algorithms for sorting, searching, and traversal. Emphasis is placed on analysing algorithmic complexity using Big-O notation, evaluating performance trade-offs, and selecting appropriate structures for real-world problems. On completion, students will be equipped to reason analytically about computational efficiency and apply these skills across subsequent software engineering modules.
Area 1 — Section 2 of 2
Learning Outcomes
AI Draft
On completion of the module, students should be able to: 2.1 explain the properties and use cases of arrays, linked lists, stacks, and queues [B, D] 2.2 analyse time and space complexity using Big-O, Big-Ω, and Big-Θ notation [B] 2.3 implement and compare sorting algorithms across different data sets [A, B] 2.4 apply binary search trees, AVL trees, and heaps to organise data efficiently [A, C] 2.5 apply graph representations and traversal algorithms to network problems [A, B] 2.6 design and justify appropriate data structure choices for software engineering scenarios [A, B, C]

Area 2 — Section 2 of 2
Teaching & Learning Strategies
AI Draft
Case-based & Problem-based LearningRecommended
Students engage with realistic computational problems drawn from industry — applying data structure knowledge to design, implement, and evaluate solutions. Develops analytical reasoning and the ability to justify design decisions using complexity evidence.
Technology-enabled & Blended LearningRecommended
Recorded micro-lectures, interactive algorithm visualisation tools (VisuAlgo), auto-graded online exercises, and LMS discussion boards support flexible, self-paced learning. Pre-class preparation shifts in-class time toward problem-solving and discussion.
Collaborative Learning ActivitiesRecommended
Group lab sessions and case-study workshops where students collaboratively analyse datasets, compare algorithm performance, and present findings — developing communication and cross-functional teamwork skills alongside technical competence.
Simulation & Lab PracticalsRecommended
Guided lab sessions where students implement each major data structure from scratch in Python, benchmark performance, and reflect on observed vs. theoretical complexity — building deep experiential understanding.
Discussion & Reflective ActivitiesRecommended
Weekly discussion prompts on the LMS forum invite students to reflect on design trade-offs, real-world algorithm use cases, and current industry practices — reinforcing metacognition and communication competencies.
AI-enabled Learning ExperiencesNew
Students use AI tools (e.g., GitHub Copilot, ChatGPT) under structured guidance to explore alternative implementations, debug code, and generate test cases — developing AI-literacy and critical evaluation of AI-generated output. Week 11 includes a dedicated AI-assisted exploration session.

Assessment Plan

Continuous Assessment (100%): Lab Assignments ×4 — 40% Four graded exercises covering arrays, sorting, trees, and graphs. Individual submission with code and written complexity justification. Mapped to outcomes 2.1–2.5. Class Test — 30% Written test on data structures theory and algorithm analysis. Closed-book, 1.5 hours. Mapped to outcomes 2.1–2.4.
